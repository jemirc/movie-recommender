# M4 시연 시나리오

## 시연 목표

10분 발표 안에서 기존 M3 추천 시스템이 정상 동작한다는 점과 M4 확장 기능인 장르 필터, 통계, UI/UX 개선을 한 번에 보여준다.

## 시연용 데이터

- 사용자: 10명
- 영화: 20편
- 평점: 80건
- 추천이 잘 보이는 사용자: `1, 김민준`
- 장르 필터 예시: `액션`

## 전체 입력 순서

아래 입력을 그대로 사용하면 기본 추천, 장르 필터 추천, 통계를 순서대로 확인할 수 있다.

```text
9
1
10
1
액션
11
1
2
3
5
0
0
```

## 단계별 시연 멘트

### 1. 프로그램 실행

```bash
make run
```

멘트:

> 프로그램을 실행하면 현재 데이터 수가 상단에 표시됩니다. 지금은 영화 20편, 사용자 10명, 평점 80건이 준비되어 있습니다.

### 2. 기존 추천 기능

입력:

```text
9
1
```

멘트:

> 먼저 M3에서 구현한 사용자 기반 추천 기능입니다. 사용자 1번 김민준의 평점과 다른 사용자들의 평점을 비교해 아직 보지 않은 영화를 추천합니다.

기대 결과:

- 추천 결과 5편 출력
- 예시 추천 영화: `밀수`, `헤어질 결심`, `암살`, `어바웃 타임`, `범죄도시`
- 추천 점수 예시: `밀수 474.75`

### 3. 장르 필터 추천

입력:

```text
10
1
액션
```

멘트:

> 이번에는 확장 기능인 장르 필터 추천입니다. 같은 사용자에게 추천하되, 액션 장르만 남겨 결과를 좁힙니다. 입력 전에 사용 가능한 장르와 영화 수가 표시됩니다.

기대 결과:

- `액션 추천 결과: 3편`
- 예시 추천 영화: `암살`, `범죄도시`, `베테랑`

### 4. 통계 기능

입력:

```text
11
1
2
3
5
0
```

멘트:

> 통계 메뉴에서는 전체 평균 평점, 장르별 통계, 평점 Top N 영화를 확인할 수 있습니다. 발표에서는 Top 5를 보여주겠습니다.

기대 결과:

- 전체 평균 평점: `4.36`
- 전체 평점 수: `80건`
- 인기 장르: `드라마 (18건)`
- Top 5 예시: `듄`, `서울의 봄`, `범죄도시`, `헤어질 결심`, `극한직업`

### 5. 종료

입력:

```text
0
```

멘트:

> 종료 시 CSV 파일로 현재 데이터를 저장합니다. 파일 입출력 예외는 main에서 한 번 더 처리해 저장 실패도 확인할 수 있습니다.

## 리허설 체크

- 전체 입력 흐름 정상 종료 확인
- 메뉴 `1`부터 `11`까지 임시 데이터로 리허설 완료
- 원본 CSV가 손상되지 않도록 추가/수정 테스트는 임시 복사본에서 진행
- GitHub `main`에 최종 push 완료 후 발표 전 한 번 더 `make run` 확인
